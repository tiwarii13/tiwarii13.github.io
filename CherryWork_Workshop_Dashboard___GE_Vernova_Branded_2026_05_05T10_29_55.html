Picking — Current vs Future

Current (Paper-Based)

  • 1. Supervisor uses LB13 on PC — SAP transaction LB13 used to create Transfer Orders from Transfer Requirements linked to production/work orders
  • 2. Creates TOs from Transfer Reqs — TOs generated from open Transfer Requirements; movement types 319, 201, Z21/Z22 used
  • 3. Prints: cover + pick list + slips — Multiple paper documents printed: cover page, picking list, and individual pick slips per line item
  • 4. Assigns papers to workers — Physical paper documents distributed to warehouse operators for manual execution
  • 5. Workers pick manually — Operators navigate to bins, pick materials onto trolley based on printed instructions
  • 6. Returns papers for confirmation — Completed pick slips returned to supervisor for manual TO confirmation in SAP
VS

Future (CherryWork Mobile)

  • 1. Supervisor creates TOs (same) — TO creation process unchanged; supervisor still uses LB13 or similar
  • 2. Picker scans TO on mobile — Worker scans TO number barcode on Honeywell device; must use TO numbers, NOT production order numbers
  • 3. Sees all line items on screen — All pick lines displayed digitally; auto-selection of line items to reduce clicks
  • 4. Scans bin barcode for verification — Storage type + bin barcode scanned to verify correct location before picking
  • 5. Confirms each line on device — One-by-one line item processing with real-time confirmation; eliminates paper trail
  • 6. Only cover page printed — Eliminates individual pick slips; only cover page and picking list printed for reference
Must use TO numbers, NOT production order numbers — earlier CherryWork versions only supported PO; now updated for TOs — this is critical for AHT's production order picking workflow

Additional Processes

Bin-to-Bin Transfer

  • Rarely used at AHT — Only needed when suggested put-away bin is full; stock confirmed to suggested bin first, then moved
  • LT01, mvt type 998/999 — Standard SAP transaction LT01 with movement types 998 (within warehouse) or 999 (between storage types)
  • Already in CherryWork — Supports select by bin, view materials in a bin, and move multiple materials at once with enhanced features

Fixed Bin Replenishment

  • LP21/LP24 transactions — SAP transactions LP21 (create TO for replenishment) and LP24 used for fixed bin stock management
  • For Kanban materials — Fixed bins defined in material master for Kanban items in KB1/KB2 storage types requiring periodic replenishment
  • Overflow bin issues noted — When excess material overflows the fixed bin, LP21 helps create TOs to replenish from overflow bins back to fixed locations

Special Stock (P/Q)

  • P = Project stock — Procurement against project (WBS element); shows in project-specific inventory; used for cross-plant transfers (e.g., plant 6048 to 3247)
  • Q = Consumption on receipt — Material consumed immediately upon goods receipt; no warehouse stock created
  • Not yet in CherryWork — API supports passing these indicators but UI display not yet implemented; flagged as a requirement for display-only visibility

Key Requirements Identified

1. Auto Movement Type Determination
Based on incoterms, lead time, account assignment — parameterized per plant/system (AHT vs Fusion vs Appsys); origin acceptance enabled leads to only 107/109; otherwise 101, 105, or 107
2. Editable Document Date
KPI: truck unload to MIGO posting time — defaults today, must be changeable; critical for measuring warehouse receiving efficiency
3. Month-End Posting Controls
Replicate custom SAP table ZTMM0080 logic in CherryWork — auto-date logic and enhancement exits need validation against standard APIs
4. Storage Type + Bin Verification
Must use combined field (e.g., FKG-D61), not bin ID alone — same bin can exist across different storage types; using bin ID alone would cause misplacement
5. Special Stock Indicators (P/Q)
Display-only — API supports it but not yet implemented in UI; P = project stock (WBS-driven), Q = consumption on receipt; does not change workflow
6. Custom PO Filtering
Exclude stale orders from 2011/2012 in open PO bucket list — standard product assumes orders are closed on time; custom logic needed for AHT
7. Batch Management
~30-40 materials — free-text vendor ID with duplicate risk from spelling variations; used to ensure picking consistency (e.g., all insulators from same supplier)
8. TO Line Item Processing
One-by-one via combined TO + item barcode — auto-selection to reduce clicks; preference for streamlined UX with minimal manual interactions
